# 功能实现说明

## 1. 项目定位

本项目是面向《暗黑破坏神 IV》溶解旋风斩野蛮人的 Windows 屏幕视觉自动化原型。

程序通过屏幕截图判断游戏状态，并通过普通键盘、鼠标输入执行动作。当前实现不读取游戏内存、不注入游戏进程、不修改网络数据，也不包含反作弊绕过功能。

自动化操作可能违反游戏服务条款并带来账号风险。当前版本必须先完成实机截图采集和坐标校准，不能直接视为可无人值守运行的成品。

## 2. 系统架构

| 模块 | 文件 | 职责 |
|---|---|---|
| 配置加载 | `src/d4bot/config.py` | 加载 YAML，校验运行模式和归一化坐标 |
| 屏幕采集 | `src/d4bot/capture.py` | 使用 MSS 捕获指定显示器 |
| 视觉识别 | `src/d4bot/vision.py` | 判断生命、敌人、掉落、背包界面和背包占用率 |
| 输入控制 | `src/d4bot/input.py` | 执行键鼠输入并实施窗口、安全开关检查 |
| 战斗控制 | `src/d4bot/combat.py` | 管理先祖召唤、旋风斩、冷却技能、药水和撤退 |
| Boss 流程 | `src/d4bot/boss.py` | 移动到召唤装置、角落和宝箱并执行交互 |
| 拾取控制 | `src/d4bot/loot.py` | 通过强制交互键逐次拾取物品 |
| 背包控制 | `src/d4bot/inventory.py` | 打开背包、回城、仓库存放和返回传送门 |
| 方向控制 | `src/d4bot/route.py` | 旋风斩期间循环移动鼠标指针控制方向 |
| 状态机 | `src/d4bot/engine.py` | 编排完整屠夫循环和异常保护 |
| 命令入口 | `src/d4bot/cli.py` | 提供运行和校准命令 |

## 3. 已实现功能

### 3.1 安全控制

- 默认使用 `dry_run`，动作仅记录日志，不发送键鼠输入。
- 只有前台窗口标题包含 `Diablo IV` 时才允许输入。
- 窗口失焦时自动暂停。
- `F8` 启用或关闭输入。
- `F9` 暂停或恢复。
- `F12` 紧急停止，停止后必须重启程序。
- 输入被拦截或执行失败时不推进相关状态。
- 状态切换时可自动保存截图到 `runtime/`。

### 3.2 视觉识别

| 能力 | 实现方式 | 当前限制 |
|---|---|---|
| 生命值估算 | 统计生命区域红色 HSV 像素比例 | 必须按实际 HUD 校准 |
| 周围敌人检测 | 统计角色周围红色敌对像素 | 可能受场景、特效和血条颜色影响 |
| 掉落提示检测 | 检测亮色、低饱和度标签像素 | 可能把其他白色 UI 或场景识别为掉落 |
| 背包界面检测 | 统计背包区域亮度 | 仓库和背包界面可能需要分别识别 |
| 背包占用估算 | 将区域划分为 `4×10`，检测每格内部边缘密度 | 阈值必须用空包、满包截图校准 |

### 3.3 野蛮人战斗

当前按键配置：

| 动作 | 按键 |
|---|---|
| 移动和普通交互 | 鼠标左键 |
| 强制交互 | `F` |
| 旋风斩 | 鼠标右键 |
| 先祖召唤 | `1` |
| 挑战怒吼 | `2` |
| 战吼 | `3` |
| 集结呐喊 | `4` |
| 钢铁之肤 | `5` |
| 药水 | `Q` |
| 闪避 | `Space` |

战斗行为：

- 每次 Boss 战开始时释放一次先祖召唤。
- 使用引导切换模式开启或关闭旋风斩。
- 开启旋风斩后等待 `restart_delay`，再释放其他技能。
- `2` 至 `5` 根据配置的冷却时间和优先级释放。
- 旋风斩期间循环移动鼠标指针控制行进方向。
- 生命低于喝药阈值时按 `Q`，默认限制为每秒一次。
- 生命低于撤退阈值时停止旋风斩、闪避并尝试回城。

技能冷却目前由固定秒数估算，尚未识别技能栏图标的实际冷却状态。

### 3.4 屠夫循环

状态机已实现以下流程：

1. 移动到召唤装置。
2. 等待人物移动完成。
3. 按 `F` 召唤 Boss。
4. 等待 `1.5` 秒。
5. 移动到 Boss 房角落。
6. 等待移动时间和额外 `3` 秒。
7. 释放先祖召唤并开启旋风斩。
8. 检测 Boss 是否出现。
9. 战斗期间控制旋风斩方向并释放冷却技能。
10. Boss 曾经出现且连续确认周围无怪后，认定战斗结束。
11. 移动到宝箱。
12. 等待人物移动完成。
13. 按 `F` 打开宝箱。
14. 等待掉落出现并逐帧拾取。
15. 连续多帧看不到掉落后打开背包。
16. 判断背包是否已满。
17. 未满则开始下一轮召唤。
18. 已满则进入回城和仓库流程。

`F7` 可以请求重新开始召唤流程。

### 3.5 拾取逻辑

- 只有战斗结束后才进入宝箱和拾取阶段。
- 拾取期间再次检测到敌人时立即暂停。
- 每次只发送一次 `F`，下一帧重新判断掉落是否仍存在。
- 连续达到 `loot_clear_frames` 帧没有掉落提示后结束拾取。
- 达到最大拾取次数后仍检测到掉落时暂停，避免无限循环。

### 3.6 背包和仓库

- 打开背包后确认界面是否实际出现。
- 以 `4×10` 网格估算背包占用率。
- 默认 `full_ratio: 1.0`，只有检测为全满才回城。
- 回城后按配置坐标移动到仓库。
- 等待移动完成后按 `F` 打开仓库。
- 仓库界面未出现时暂停。
- 只对视觉判定为有物品的格子执行 `Shift+左键`。
- 转移后重新检测背包占用率。
- 背包未降到 `empty_ratio` 以下时暂停。
- 存放成功后关闭仓库，移动到返回传送门并按 `F`。
- 等待加载完成后开始下一轮。

当前没有物品价值判断、收藏标记识别、仓库容量判断或仓库分页管理。

### 3.7 输入后端

| 模式 | 行为 |
|---|---|
| `dry_run` | 记录动作，不发送输入 |
| `native` | 脚本直接发送普通键鼠输入 |
| `macro_hotkey` | 战斗中触发配置的鼠标宏快捷键 |

推荐先使用 `dry_run` 完成全部识别和坐标校准，再考虑 `native`。

## 4. 状态机

主要状态如下：

```text
OBSERVE
  -> SUMMON_BOSS
  -> SUMMON_TRAVEL
  -> INTERACT_SUMMON
  -> POST_SUMMON_WAIT
  -> MOVE_CORNER
  -> CORNER_TRAVEL
  -> CORNER_WAIT
  -> COMBAT
  -> MOVE_CHEST
  -> CHEST_TRAVEL
  -> OPEN_CHEST
  -> WAIT_CHEST_LOOT
  -> LOOT_CHEST
  -> OPEN_INVENTORY
  -> WAIT_INVENTORY
```

背包未满：

```text
WAIT_INVENTORY -> SUMMON_BOSS
```

背包已满：

```text
WAIT_INVENTORY
  -> RETURN_TOWN
  -> WAIT_TOWN
  -> MOVE_STASH
  -> STASH_TRAVEL
  -> OPEN_STASH
  -> WAIT_STASH
  -> DEPOSIT_STASH
  -> VERIFY_DEPOSIT
  -> MOVE_RETURN_PORTAL
  -> PORTAL_TRAVEL
  -> USE_RETURN_PORTAL
  -> WAIT_RETURN
  -> SUMMON_BOSS
```

异常状态：

```text
任意活动状态 -> PAUSED
严重低血 -> RETREAT -> PAUSED
F12 -> STOPPED
```

## 5. 配置文件

- `config/default.yaml`：运行、视觉、职业、拾取、背包、仓库和方向配置。
- `config/bosses/butcher.yaml`：屠夫召唤、角落、宝箱和战斗时序。

所有屏幕位置使用归一化坐标：

```text
[0.0, 0.0] = 屏幕左上角
[1.0, 1.0] = 屏幕右下角
[0.5, 0.5] = 屏幕中心
```

## 6. 自动暂停条件

以下情况会暂停流程：

- 游戏窗口失焦。
- Boss 在等待时间内从未被检测到。
- 拾取期间检测到敌人。
- 达到拾取次数上限后仍有掉落。
- 背包界面未打开。
- 仓库界面未打开。
- 背包中没有识别到可转移物品。
- 仓库存放后背包仍未清空。
- 战斗输入被安全门阻止。
- 无法停止旋风斩或执行回城。
